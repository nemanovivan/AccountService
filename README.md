## AccountService
## Серверная часть AccountServic'а
### БД
Для того чтобы работала, надо поднять базу данных и в application.properties указать её url, username и password. Скрипт для создания базы данных лежит в /src/main/resources/db
### Лог
Лог выводится по Post-запросу на апи "/log" в файл src/main/resources/log/log.txt. Представляет собой данные о вызове addAmount, getAmount и суммарное количество вызовов. 
Также отображает количество вызовов каждого метогда в каждую минуту с начала работы сервера, если запросы были, если запросов не было, то данные за это время не выводятся.
По Delete-запросу на апи "/log" очищает файл с логом и данные
### Кэширование
Т.к. данные постоянно меняются, то кэширование реализовано следующим образом: Метод getAmount при первом обращении для каждого id берёт данные из БД и записывает их в кэш. при дальнейшем обращении данные берутся из кэша. раз в 30 секунд кэш сбрасывается и значение снова берётся из БД. То есть данные для метода getAmount при повторных обращениях обновляются раз в 30 секунд независимо от того, сколько раз был вызван метод addAmount для id
### Ошибка
Если указанного id не существует, то кидает в консоль сообщение об ошибке, что аккаунт с таким id не найден, на клиент же уходит статус-код 500
